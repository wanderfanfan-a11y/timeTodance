## 1. 项目概述

tdance 是一款轻量级、**基于 Windows 桌面**的**多定时器提醒工具**。用户可以创建多个定时器（一次性或周期性），到点后通过**系统声音 + 系统通知**提醒用户去做某件事。应用支持**开机自启**与**最小化到系统托盘**后台常驻运行，做到"设置一次、长期静默守护"。

### 1.1 核心价值

- **多任务提醒**：同时管理多个互不干扰的提醒（喝水、久坐提醒、吃药、会议、番茄钟等）。
- **零打扰常驻**：后台托盘运行，不占用任务栏与视觉空间。
- **开箱即用**：基于 Flutter 一套代码开发，Windows 首发，后续可扩展其他平台。
- **完全脱机**：数据保存在本地 SQLite（drift），无需后端服务、无需联网、无账号登录。
- **秒级精度**：触发时间精确到秒，适用番茄钟等场景。

## 2. 目标与范围

### 2.1 目标（本期要做）

- 定时器的增删改查与启用/禁用。
- 支持一次性与多种周期性触发规则。
- 到点通过系统通知 + 声音提醒。
- 桌面端：开机自启、最小化到托盘、后台常驻。
- 本地持久化，重启后定时器不丢失。

### 2.2 非目标（本期不做，列入未来规划）

- 移动端（Android / iOS）适配。
- macOS / Linux 桌面适配。
- 云同步 / 多设备账号体系。
- 团队协作、共享提醒。
- 复杂的日历视图与日程管理。
- 语音输入 / AI 智能建议。

## 3. 用户使用场景

- **久坐提醒**：每 45 分钟提醒起身活动一次（周期性·按间隔）。
- **喝水提醒**：工作日 9:00–18:00 每小时提醒（周期性·按每日时段）。
- **吃药提醒**：每天 8:00、20:00 各一次（周期性·按每日固定时刻）。
- **会议提醒**：某天某时刻一次性提醒（一次性）。
- **番茄钟**：从现在起 25 分钟后提醒（倒计时·一次性）。
- **周期例会**：每周一、三、五 10:00 提醒（周期性·按星期）。

## 5. 非功能需求

- **性能**：常驻内存占用尽量低；空闲时无明显 CPU 占用；定时触发误差 ≤ 1 秒。
- **可靠性**：应用意外退出/系统重启后，开机自启 + 配置恢复保证定时器继续生效。
- **易用性**：新建一个定时器不超过 3 步；关键操作有明确反馈。
- **兼容性**：Windows 10/11 优先；Flutter 代码结构保留扩展其他桌面的可能性。
- **可维护性**：清晰的分层架构（UI / 状态 / 领域 / 数据），核心调度逻辑与平台特性解耦。
- **隐私**：全部数据仅本地存储，不上传网络。

## 8. 技术架构与依赖选型（Flutter）

### 8.1 分层架构

- **UI 层**：Flutter Widgets（定时器列表页、编辑页、设置页）。
- **状态管理**：`riverpod`（或 `provider`），推荐 `riverpod`。
- **领域层**：定时器模型、触发规则计算、"下一次触发时间"引擎（纯 Dart，可单元测试）。
- **数据层**：本地持久化，采用 **drift + SQLite**（完全脱机，无后端 / 无网络依赖）。
- **平台服务层**：通知、声音、托盘、开机自启、窗口管理的封装（按平台条件编译/分支）。

### 8.2 候选插件（pub.dev）

| 能力 | 候选插件 | 备注 |
| --- | --- | --- |
| 窗口管理（隐藏到托盘、拦截关闭） | `window_manager` | Windows |
| 系统托盘 | `tray_manager` | Windows |
| 开机自启 | `launch_at_startup` | Windows |
| Windows 桌面通知 | `local_notifier` | 使用系统通知提醒用户 |
| 声音播放 | `audioplayers` 或 `just_audio` | 播放内置/系统提示音 |
| 本地存储（轻量） | `shared_preferences` | 设置项 |
| 本地存储（结构化） | `drift` | 定时器与设置数据，**类型安全 SQLite ORM**，便于单元测试 |
| 时区处理 | `timezone` | 通知精确调度所需 |
| 唯一 ID | `uuid` | 生成定时器 id |
| 路径 | `path_provider` | 数据/资源路径 |

> 插件选型面向 Windows 桌面，实际集成时以 `tray_manager`、`local_notifier`、`launch_at_startup` 在 Windows 10/11 的可用性为准。

### 8.3 调度引擎设计要点

- 采用**"计算下一次触发时间 + 定时唤醒"**模型：领域层负责根据规则计算 `nextTriggerAt`；调度器在最近的触发点唤醒并执行提醒，然后重新计算下一次。
- 桌面前台可用 Dart `Timer`；应对长休眠需在唤醒/启动时**对齐当前时间并补算错过的触发**。

## 9. UI / 交互设计（概要）

- **主界面（定时器列表）**：卡片列表，每项显示名称、类型标签、下一次触发时间、启用开关；顶部"新建"按钮；空状态引导。
- **新建/编辑页**：名称、类型选择（分段控件）、按类型动态展示规则输入、提醒方式（通知/声音/弹窗开关 + 铃声/音量）、贪睡设置。
- **设置页**：开机自启、关闭行为、默认提醒、勿扰时段、主题、导入导出。
- **托盘菜单（Windows）**：显示主界面 / 快速新建 / 全部暂停 / 退出。
- **提醒呈现**：系统通知 + 声音；前台时可弹出对话框（含"知道了 / 稍后提醒"按钮）。
